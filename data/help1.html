    Logging

    Es sind zwei Arten des Loggings vorgesehen:
    1. Logging via Konsole
    Hier ist das Logging auf eine feste Anzahl von Zeilen begrenzt, danach überschreibt jede neue Zeile eine alte. Folgende Themen und Inhalte sind mittels Einstellung wählbar.
    1.a RF24 (nur wenn der RF24 Gateway aktiviert ist)
    Hier werden die Eingangs- und Ausgangsnachrichten protokolliert.
    1.b Sensor
    Alle Infos zum Sensor (Initialisierung, Werte)
    1.c MQTT
    Alle MQTT Nachrichten (ein- und ausgehend)
    1.d Web Kommando
    Alle Web-Kommandos (entweder über Konsole eingegeben oder interaktiv über die Weboberfläche)
    1.e System Informationen
    Informationen zum laufenden System
    2. Logging via Datei /debugfile.txt
    2.a System Informationen
    Alle kritischen Informationen zum System plus einmal in der Stunde ein "Alive" Eintrag.
    AchtungDiese Einträge werden direkt in den SYstemspeicher des ESP geschrieben. Es besteht die Gefahr das das Filesystem volläuft und durch die dauerhafte Nutzung verschleißt der EEPROM.

    Loggingeinstellungen können temporär eingestellt werden, d.h. sie gelten maximal bis zum nächsten reboot. Alternativ können sie im EEPROM gespeichert werden und gelten dann bis sie das nächste mal geändert werden.
